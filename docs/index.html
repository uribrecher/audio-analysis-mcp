audio-analysis-mcp

A Python MCP server that imports, separates, analyzes, transcribes, and compares audio — the analysis tools an AI agent uses to figure out what a sound is made of.

★ View on GitHub Read the docs
What it does

Hear what a sound is made of

Separates stems with Demucs, profiles tone (spectrum, ADSR, modulation), transcribes notes with Basic Pitch, isolates and compares — so the agent can configure synths to match a target. GPL-3.0 (it builds on GPL components like vmo and cvxopt).

Import & render
import_audio, audio_render, audio_list_devices
Stem separation
stem_separate (Demucs)
Spectral analysis
spectrum_analyze — mel spectrogram, spectral features, ADSR, modulation
Transcription
note_transcribe (Basic Pitch → MIDI), note_triage, note_isolate
Comparison
audio_compare — target vs. synthesized: mel distance, per-band energy
FastMCP (stdio)
Python 3.11, pinned for Basic Pitch's CoreML/TensorFlow stack.
Quickstart

Up and running

# install dependencies
uv sync --dev
brew install portaudio        # required for audio_render (macOS)

# run the MCP server (stdio)
uv run python -m audio_analysis_mcp

# tests
uv run pytest -v
In the pipeline

Where it fits

Audio→Analysis→Agent→MIDI→Synth

audio-analysis-mcp is the ears — it turns raw audio into the features the agent reasons about.
